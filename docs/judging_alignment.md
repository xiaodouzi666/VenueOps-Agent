# Judging Alignment

## Technological Implementation

- FastAPI backend and Next.js web app.
- Google Cloud Run deployment assets.
- Google ADK agent definition for Agent Builder / Agent Platform alignment.
- Gemini / Vertex AI planner and explanation hooks.
- MongoDB Atlas-compatible repository layer.
- MongoDB MCP Server tool trace for `find`, `aggregate`, `count`, and schema, with transport displayed in the UI.
- MongoDB Aggregation-style risk analysis.
- Atlas Vector Search-first SOP retrieval with deterministic local fallback ranking.
- Human-in-the-loop action execution and audit log.

## Design

- First screen is an operational command center.
- Heatmap, KPI cards, agent mission, action queue, tool trace, and SOP evidence are visible without navigation.
- Approval cards include Why now, Data used, Risk, Expected impact, Approve, and Reject.

## Potential Impact

- Applies to World Cup fan zones, stadiums, malls, airports, and convention centers.
- Targets crowd safety, queue reduction, inventory availability, incident SLA, and tenant revenue.

## Quality Of Idea

- Combines event venue operations and brick-and-mortar retail.
- Uses MongoDB as operational database, agent memory, analytics engine, and semantic retrieval layer.
- Emphasizes controlled actions instead of unconstrained automation.
